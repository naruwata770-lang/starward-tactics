# レビューチェックリスト

レビュー時にこの一覧を上から順に確認する。各項目について「該当なし」「OK」「指摘あり」のいずれかを判定する。

## 正確性

- [ ] 全ての分岐 (if/else, switch, 三項演算子) に意図がある
- [ ] null / undefined / 空配列 / 空文字 を意識して扱っている
- [ ] async/await の例外は try-catch または `.catch` で受けている
- [ ] `Promise.all` で 1 つでも reject すると全体が落ちる挙動を許容しているか
- [ ] 配列のインデックスアクセスは範囲チェックされている
- [ ] 型アサーション (`as`) を使った箇所は実行時に保証があるか
- [ ] テスト (ユニット / 結合) が追加・更新されているか
- [ ] 既存テストを壊していないか

## セキュリティ

- [ ] ユーザー入力をそのまま `innerHTML` / `dangerouslySetInnerHTML` に渡していない
- [ ] URL / ファイルパスの組み立てに入力値を直接連結していない
- [ ] API キー、トークン、パスワードがコード・ログ・コミットに含まれていない
- [ ] 認可チェック (誰がこの操作をできるか) が抜けていない
- [ ] 外部 URL への遷移は `rel="noopener noreferrer"` を付けている
- [ ] 新規依存パッケージは信頼できる発行元か、既知脆弱性は無いか
- [ ] localStorage / sessionStorage に機密情報を保存していない

## 保守性

- [ ] 関数 / コンポーネント名から役割が即座に分かる
- [ ] 1 関数の行数・複雑度が過剰でない (目安: 50 行以内、ネスト 3 段以内)
- [ ] マジックナンバー / マジックストリングが定数化されている
- [ ] 同じロジックの重複が無い (DRY)
- [ ] コメントは「なぜそうしたか」を説明している (「何をしているか」だけのコメントは不要)
- [ ] 不要な `console.log` / デバッグコードが残っていない
- [ ] dead code (使われていない関数・変数・import) が無い
- [ ] TODO / FIXME に追跡可能な情報 (Issue 番号など) が付いている

## パフォーマンス

- [ ] React コンポーネントの再レンダリング条件が妥当
- [ ] `useEffect` / `useMemo` / `useCallback` の依存配列が正しい
- [ ] 大量データのレンダリングは仮想化 / 分割を検討した
- [ ] イベントハンドラ (resize, scroll, input) は必要に応じてデバウンス / スロットル
- [ ] 重い計算は `useMemo` / Web Worker 等で隔離されている
- [ ] バンドルサイズが大幅に増える依存を追加していない
- [ ] 画像・アセットは適切に最適化 / 遅延読み込みされている

## 一貫性

- [ ] ESLint / Prettier がパスする
- [ ] 既存の命名規則 (camelCase, PascalCase 等) に従っている
- [ ] ファイル配置 (`src/components`, `src/hooks` など) が既存パターンに合っている
- [ ] 既にあるユーティリティ・コンポーネントを再利用している (車輪の再発明をしていないか)
- [ ] import 順序が既存ファイルと揃っている
- [ ] 型定義のスタイル (`interface` vs `type`) が既存と揃っている
- [ ] Tailwind クラスの並び順が既存ファイルと揃っている
- [ ] コミットメッセージが既存の規約に従っている
